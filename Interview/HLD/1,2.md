### âœ… **1. What is High Level Design (HLD)?**

**High Level Design (HLD)** is the **blueprint** of the system. It explains the system's **overall architecture**, major components, how they interact, and the **technology stack** to be used. It focuses on **what** needs to be built rather than **how** it will be built at the code/class level.

#### **Key Points About HLD**

| Feature         | Description                                                        |
| --------------- | ------------------------------------------------------------------ |
| Focus           | Overall system structure & modules                                 |
| Audience        | Architects, Tech Leads, Senior Developers                          |
| Contains        | Architecture diagrams, technology choices, integration points      |
| Level of Detail | Broad, conceptual                                                  |
| Purpose         | To understand system behavior and validate architectural decisions |

---

### âœ… **2. What is Low Level Design (LLD)?**

**Low Level Design (LLD)** breaks the high-level architecture into **detailed component and class-level designs**. It focuses on **how** to implement the actual logic.

#### **Key Points About LLD**

| Feature         | Description                                                   |
| --------------- | ------------------------------------------------------------- |
| Focus           | Implementation details, class diagrams, function-level design |
| Audience        | Developers                                                    |
| Contains        | Database schema, class diagrams, APIs, methods, pseudocode    |
| Level of Detail | Concrete & specific                                           |
| Purpose         | To guide actual coding and development                        |

---

### âœ… **Difference Between HLD and LLD**

| Criteria     | HLD                                        | LLD                                             |
| ------------ | ------------------------------------------ | ----------------------------------------------- |
| View         | Birdâ€™s eye system view                     | Detailed internal component view                |
| Audience     | Architects & Leads                         | Developers                                      |
| Contains     | Architecture diagrams, modules, tech stack | Class diagrams, database tables, function logic |
| Detail Level | Abstract                                   | Detailed with actual process flow               |
| When Created | During early design phase                  | After HLD is approved                           |
